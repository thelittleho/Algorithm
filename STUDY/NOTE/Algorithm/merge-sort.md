#  합병 정렬(Merge Sort) 핵심 요약

## 1. merge_sort 함수: "끝까지 쪼개기" (Divide)
`merge_sort` 함수는 배열을 더 이상 나눌 수 없을 때까지(원소가 1개일 때까지) 반으로 쪼개는 역할을 합니다.


### 💡 동작 과정 (예: [38, 27, 43, 3])
1. **반으로 나누기**: 중간 지점(`mid`)을 계산해 왼쪽(`38, 27`)과 오른쪽(`43, 3`)으로 나눕니다.
2. **재귀 호출**: 나뉜 덩어리들을 다시 `merge_sort`에 넣습니다.
   - `38, 27` → `38`과 `27`로 쪼개짐
   - `43, 3` → `43`과 `3`으로 쪼개짐
3. **바닥 찍기**: 원소가 하나가 되면(`start >= end`), "나 다 쪼개졌어!" 하고 위로 보고(`return`) 합니다.

## 🤝 2. _merge 함수: "순서대로 합치기" (Conquer)
진정한 정렬은 여기서 일어납니다. 이미 정렬된 두 팀을 비교하며 하나의 완성된 팀으로 만듭니다.

### 💡 합치는 전략: "포인터 게임"
- **왼쪽 팀 포인터(i)**와 **오른쪽 팀 포인터(j)**가 각 팀의 맨 앞 선수를 가리킵니다.
- 둘을 비교해서 **더 작은 선수**를 임시 배열(`sorted`)에 넣고, 그 선수가 속한 팀의 포인터만 한 칸 뒤로 옮깁니다.

[Image of merging two sorted arrays using pointers]

> **예시: [27, 38] 팀과 [3, 43] 팀이 만났을 때**
> 1. `27` vs `3` → `3`이 더 작으므로 `sorted`에 추가, 오른쪽 포인터 이동.
> 2. `27` vs `43` → `27`이 더 작으므로 `sorted`에 추가, 왼쪽 포인터 이동.
> 3. `38` vs `43` → `38`이 더 작으므로 `sorted`에 추가, 왼쪽 포인터 이동.

## 🏃‍♂️ 3. 남은 사람을 왜 그대로 붙이나요? (최적화)
이 부분이 합병 정렬이 똑똑한 이유입니다! 한 팀이 먼저 모두 `sorted` 배열로 들어갔을 때, 남은 팀원들은 비교할 필요가 없습니다.

### 💡 이유: "이미 검증된 실력자들"
1. **자기 팀 안에서**: 이미 정렬된 상태로 들어왔습니다. (재귀의 힘!)
2. **상대 팀과 비교했을 때**: 이미 상대 팀의 가장 큰 마지막 선수보다도 크기 때문에 남겨진 것입니다.
3. **결론**: 따라서 남은 선수들은 현재 `sorted`에 담긴 그 어떤 숫자보다 크며, 자기들끼리도 순서가 맞으므로 **그대로 싹 긁어서 뒤에 붙여도** 정렬이 완벽하게 유지됩니다.

```cpp
// 한쪽 팀이 다 나갔을 때, 남은 쪽을 통째로 복사하는 구간
while (i <= mid) sorted[k++] = array[i++]; // 왼쪽 팀 잔여 처리
while (j <= end) sorted[k++] = array[j++]; // 오른쪽 팀 잔여 처리
