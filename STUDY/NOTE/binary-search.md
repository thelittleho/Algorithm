# 🔍 이진 탐색(Binary Search) 핵심 요약

이진 탐색은 업다운(Up-Down) 게임과 같은 원리로, 검색 범위를 매번 절반으로 줄여나가기 때문에 매우 빠릅니다 ($O(\log n)$). 
단, **반드시 정렬된 상태**에서만 사용할 수 있습니다.

---

## 💡 동작 과정 (예: [1, 3, 5, 6, 7, 10, 16, 24, 32], 타겟: 24)

### 1단계: 초기 설정
- **시작(start)**: 0, **끝(end)**: 8, **중간(mid)**: 4 (값: `7`)
- 비교: 타겟(`24`) > 중간값(`7`) 
- **결과**: `7` 이하의 왼쪽 데이터는 모두 검색 대상에서 제외합니다.



### 2단계: 오른쪽 절반 탐색
- **시작**: `mid + 1` (인덱스 5), **끝**: 8
- 새로운 **중간(mid)**: 6 (값: `16`)
- 비교: 타겟(`24`) > 중간값(`16`)
- **결과**: `16` 이하의 데이터도 모두 제외합니다.

### 3단계: 다시 범위 좁히기
- **시작**: `mid + 1` (인덱스 7), **끝**: 8
- 새로운 **중간(mid)**: 7 (값: `24`)
- 비교: 타겟(`24`) == 중간값(`24`)
- **결과**: **탐색 성공!** 인덱스 `7`을 반환합니다.

---

## 🛠️ C++ 핵심 코드 

```cpp
int binarySearch(const vector<int>& array, int target) {
    int start = 0;
    int end = array.size() - 1;

    while (start <= end) {
        int mid = start + (end - start) / 2; // (start + end) / 2 보다 안전한 방식

        if (array[mid] == target) {
            return mid; // 찾았을 때
        }
        else if (array[mid] < target) {
            start = mid + 1; // 타겟이 오른쪽에 있을 때
        }
        else {
            end = mid - 1; // 타겟이 왼쪽에 있을 때
        }
    }
    return -1; // 못 찾았을 때
}
```
## 📌 이진 탐색(Binary Search) 요약 표

| 항목 | 내용 |
| :--- | :--- |
| **전제 조건** | 반드시 데이터가 **정렬**되어 있어야 함 |
| **시간 복잡도** | **$O(\log n)$** (매우 빠름) |
| **핵심 로직** | 중간값(`mid`)과 타겟을 비교해 범위를 절반씩 제거함 |
| **장점** | 데이터가 커질수록 선형 탐색($O(n)$)보다 압도적으로 유리함 |
