# 🚀 C++ STL Algorithm 핵심 정리

## 1. Algorithm이란?
**알고리즘(Algorithm)**은 데이터를 처리, 조작, 관리하기 위해 정의된 일련의 절차나 함수를 의미합니다.
C++에서는 **STL(`Standard Template Library`)**을 통해 강력하고 효율적인 알고리즘들을 제공하므로, 직접 모든 로직을 구현하지 않고도 최적화된 함수들을 즉시 사용할 수 있습니다.

---

## 2. 정렬 알고리즘 (Sorting)

### 📌 sort()
범위 내의 요소들을 **오름차순**으로 정렬합니다. 
STL의 `sort`는 상황에 따라 알고리즘을 변경하는 **인트로 정렬(`Introsort`)** 방식을 채택합니다.

| 구분 | 퀵 정렬 (Quick Sort) | 힙 정렬 (Heap Sort) | 삽입 정렬 (Insertion Sort) |
|:---|:---|:---|:---|
| **평균 시간 복잡도** | $O(n \log n)$ | $O(n \log n)$ | $O(n^2)$ |
| **최악 시간 복잡도** | $O(n^2)$ | $O(n \log n)$ | $O(n^2)$ |
| **주요 장점** | 실질적 속도가 가장 빠름 | 최악의 경우에도 성능 보장 | 작은 데이터에서 효율 최고 |
| **주요 단점** | 피벗에 따라 성능 저하 가능 | 퀵 정렬보다 실제 속도 느림 | 데이터가 많으면 사용 불가 |
| **정렬 방식** | 분할 정복 (Divide & Conquer) | 완전 이진 트리 (Heap) | 순차 비교 및 적정 위치 삽입 |
| **메모리 사용** | $O(\log n)$ (재귀 스택) | $O(1)$ (제자리 정렬) | $O(1)$ (제자리 정렬) |

#### 💡 STL sort (Introsort) 작동 전략
1. **Quick Sort:** 기본 전략. 평균적으로 가장 빨라 우선 실행됩니다.
2. **Heap Sort:** 재귀 깊이가 깊어지면 $O(n \log n)$ 보장을 위해 전환됩니다.
3. **Insertion Sort:** 데이터가 약 16개 이하로 적어지면 삽입 정렬로 마무리합니다.

### 📌 reverse()
지정한 범위 내 요소들의 순서를 **역순**으로 뒤집습니다.

## 3. 검색 알고리즘 (Searching)

### 📌 find()
범위 내에서 특정 값을 찾아 해당 요소의 **반복자**(`Iterator`)를 반환합니다.
- **방식:** 선형 탐색 ($O(n)$)

### 📌 binary_search()
**이진 탐색**을 사용해 값의 존재 여부를 `bool` 값으로 반환합니다.
- **조건:** 반드시 데이터가 **정렬**되어 있어야 함 ($O(\log n)$).

## 4. 기타 유용한 알고리즘 (Etc.)

| 함수명 | 설명 |
|:---|:---|
| **max_element()** | 가장 큰 값을 가리키는 반복자(`Iterator`) 반환 |
| **min_element()** | 가장 작은 값을 가리키는 반복자(`Iterator`) 반환 |
| **count()** | 특정 값과 일치하는 요소의 개수 반환 |
| **unique()** | 인접한 중복 원소 제거 (정렬 후 사용 권장) |
