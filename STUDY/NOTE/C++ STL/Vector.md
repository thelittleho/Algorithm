# 📦 C++ STL Vector 핵심 정리

## 1. Vector란?
**vector**는 C++ STL의 대표적인 **시퀀스 컨테이너**입니다.
일반 배열과 달리 **동적으로 크기를 조절**할 수 있는 스마트한 배열로, 요소들을 연속된 메모리 공간에 저장하여 관리합니다.

---

## 2. 주요 특징 (Advantages)
- **동적 크기 조절**: 데이터 개수에 따라 자동으로 메모리 공간을 늘리거나 줄입니다.
- **임의 접근(Random Access)**: 배열처럼 인덱스(`[]`)를 통해 특정 요소에 즉시 접근할 수 있습니다 ($O(1)$).
- **연속적 메모리 할당**: 데이터가 메모리상에 붙어 있어 캐시 효율이 좋습니다.
- **끝점 최적화**: 맨 뒤에 요소를 추가(`push_back`)하거나 삭제(`pop_back`)하는 작업이 매우 빠릅니다 ($O(1)$).
- **템플릿 클래스**: 템플릿 클래스이므로, 모든 데이터 타입을 사용가능합니다.

---

## 3. 단점 및 주의사항 (Disadvantages)
- **중간 삽입/삭제 성능 저하**: 배열 중간에 값을 넣거나 빼면, 뒤에 있는 모든 요소를 한 칸씩 밀거나 당겨야 하므로 성능이 떨어집니다 ($O(n)$).
- **재할당 오버헤드**: 예약된 공간(Capacity)이 꽉 차면 새로운 더 큰 메모리 공간을 찾아 전체 데이터를 복사하는 과정이 발생합니다.

---

## 4. 선언 및 기초 사용법
C++에서 `string` 상수는 반드시 **큰따옴표(`" "`)**를 사용해야 함에 주의하세요!

```cpp
#include <iostream>
#include <vector>
#include <string>

using namespace std;

int main() {
    // 1. 초기화 리스트를 이용한 생성
    vector<string> str = {"Hello", "world"};
    vector<int> num = {0, 1, 5, 6};

    // 2. 빈 벡터 생성 후 요소 추가
    vector<int> empvect;
    empvect.push_back(10); // 맨 뒤에 10 추가
    empvect.push_back(20); // 맨 뒤에 20 추가

    // 3. 크기 확인
    cout << "Size: " << empvect.size() << endl;

    return 0;
}
```
## 5. 자주 쓰는 핵심 메소드 (Member Functions)

| 분류 | 메소드 | 설명 |
|:---:|:---|:---|
| **추가/삭제** | `push_back(val)` | 맨 뒤에 원소 추가 |
| | `pop_back()` | 맨 뒤의 원소 제거 |
| | `insert(iter, val)` | 지정한 위치(`Iterator`)에 원소 삽입 |
| | `erase(iter)` | 지정한 위치(`Iterator`)의 원소 제거 |
| **데이터 접근** | `at(index)` | 해당 인덱스 접근 (범위 검사 포함) |
| | `operator[]` | 해당 인덱스 접근 (범위 검사 없음, 속도 빠름) |
| | `front()` / `back()` | 첫 번째 / 마지막 원소의 참조 반환 |
| **반복자** | `begin()` / `end()` | 시작 지점 / 끝 다음 지점의 **반복자(`Iterator`)** 반환 |
| **크기/용량** | `size()` | 저장된 원소의 개수 반환 |
| | `capacity()` | 할당된 전체 메모리 공간 크기 반환 |
| | `empty()` | 벡터가 비어있는지 여부 확인 (`bool`) |
| | `clear()` | 모든 원소 제거 (메모리는 해제되지 않음) |
| | `reserve(n)` | `n`개의 원소를 저장할 공간을 미리 예약 (성능 최적화) |
